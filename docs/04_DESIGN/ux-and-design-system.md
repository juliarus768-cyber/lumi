## Math Adventure 10 – UX & Design System Specification

Version 1.0 Premium Game Experience Guidelines

## Purpose

This document defines the visual language and interaction principles for Math Adventure. It ensures every screen feels like a premium children's adventure game while remaining educational, accessible, and consistent.

## 1. Design Vision

- Look and feel comparable to Duolingo, Pokémon Quest, and modern Nintendo interfaces.

- Never resemble a traditional worksheet or dashboard.

- Bright, optimistic, premium-quality illustrations.

- Every screen should invite exploration.

## 2. Design Principles

- One primary action per screen.

- Large touch-friendly controls.

- Minimal reading before action.

- Reward exploration with delightful animations.

- Learning should always feel playful.

## 3. Target Devices

- iPad (primary)

- Android tablets

- Desktop browser

- Future mobile phone support

## 4. Color System

| Category | Purpose | Guideline |
| --- | --- | --- |
| Primary | Buttons & actions | Bright, cheerful accent color |
| Secondary | Panels | Soft gradients and rounded cards |
| Success | Rewards | Gold, green, celebration colors |
| Warning | Hints | Warm amber, never alarming |
| Error | Input validation | Gentle red with encouraging text |

## 5. Typography

- Large headings for children.

- Readable sans-serif font.

- High contrast.

- No dense paragraphs on child screens.

- Short encouraging messages.

## 7. Animation System

- Screen transitions under 300 ms.

- Mission completion celebration.

- Pet idle animations.

- Treasure chest opening animation.

- XP and coin count-up animation.

- Reduced motion mode available.

## 8. Screen States

- Loading

- Empty

- Offline

- Error

- Mission Complete

- Tutor Mode Active

- Reward Pending

- Reward Approved

- New Badge

- Level Up

## 9. Accessibility

- Large touch targets.

- Readable text.

- Keyboard never covers question.

- Color is never the only indicator.

- Future text-to-speech support.

## 10. Tablet Layout Rules

- Landscape-first design.

- Portrait fully supported.

- No horizontal scrolling.

- Important actions remain above the fold.

- Consistent bottom navigation.

## 11. Illustration Style

- Hand-crafted, storybook quality.

- Unique themed worlds.

- Expressive pets.

- Rich backgrounds.

- Avoid generic AI-looking assets.

## 12. Sound & Haptics (Future)

- Soft reward sounds.

- Pet voice effects.

- Subtle button feedback.

- Optional background music.

- Respect system accessibility settings.

## 13. Acceptance Criteria

- Every screen follows the design system.

- Children can navigate without reading long instructions.

- Visual consistency across all modules.

- Premium game feel maintained throughout the experience.
